`modules/config` 文件每一行的含义：

1. 哈希表的数量
2. 哈希函数的数量
3. 每张图片的查询结果
4. base 文件的路径
5. query 文件的路径
6. out 文件的路径
7. 1 表示读取哈希表和哈希函数，之后跟读取路径；0 表示不读取而是重新构建哈希表
8. 1 表示存储哈希表和哈希函数，之后跟存储路径；0 表示不存储

# 哈希优化流程

- [x] 读取配置文件确定哈希参数，避免改动代码造成的重新编译
- [x] 保存哈希函数、哈希表
- [x] 不均匀哈希改进
- [x] 读取哈希函数、哈希表，直接查询
- [x] 优化性能，空间换时间，从文件哈希时，取消每次重复的读文件，哈希时间从 5000 秒降低至 250 秒
- [ ] 多进程处理多表查询时，去除重复结果
- [ ] 多进程哈希
